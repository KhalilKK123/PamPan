Legacy application
The existing Flutter application under pam_pan/.
Disposition
An evidence-backed classification: Implemented, Partial-WIP, Dead-Abandoned, Inferred, or Unknown.
Reachable screen
A UI screen that I1 can trace from the application entry point or another reachable route/action.
Parity requirement
A numbered rewrite requirement traceable to legacy evidence; it is not a target-stack decision.
I1
Repository reverse-engineering and rewrite specification increment for PAMPAN-REWRITE-SPEC.
