System overview

Type: reference · Authority: verified repository-derived behavior and repository operating boundary · Status: I1A verified; larger native I1 runtime verification blocked · Updated: 2026-07-21

PamPan is a public repository containing a legacy Flutter pantry-management application at pam_pan/ and a native foundation under apps/. Android is the sole active native application. The SwiftUI/Xcode foundation at apps/ios/ is preserved dormant and excluded from active implementation and gates. I1A verified this scope and its public-repository safety enforcement without adding product behavior.

Repository-observed native foundation

The active Kotlin/Jetpack Compose/Gradle foundation has Android-only CI and native-foundation validation, pinned Build Tools 36.0.0, strict dependency verification and locking, static/unit checks, assembled-boundary auditing, generated-APK scanning, and an offline emulator/connected-test path. supabase/ and contracts/ remain reserved-only documentation. The larger I1 has no successful emulator launch, connected-test result, TalkBack result, or actual GitHub Actions run and remains BLOCKED.

Public-repository security

The implemented public-repository security boundary scans publishable working-tree and staged-index paths/content, reachable Git objects, ref names, and requested APK metadata/content with opaque redacted output. Exact approved placeholders and publishable legacy Google client configuration are classified by digest only. I1A verification and review passed for its recorded state; later state changes require fresh evidence.

Observed implemented boundaries

  • Presentation: Flutter widgets in pam_pan/lib/. The root path initializes notifications/Firebase and attempts an Appwrite session before mounting Home or a connection-error widget (lib/main.dart:19-53).
  • Persistence/auth: the active client-side path uses Appwrite account, database-document, and realtime APIs in lib/backend/appwrite_client.dart. Appwrite server schema, permissions, ownership, and live availability are unknown.
  • External clients: source contains Firebase Vertex AI wrappers, Open Food Facts HTTP lookup, Maps/location/directions code, local-notification helpers, camera/barcode code, and an iOS Home Widget helper. Their runtime availability and complete workflows are unverified.
  • Legacy alternative: lib/backend/libdb.dart is an uncalled SQLite implementation; static analysis classifies it as dead/abandoned, not active persistence.

Observed legacy behavior

Static reachability from Home includes pantry item create/edit/list/filter/delete, category management, an expiry calendar, a fixed-sample recipe response, mock card validation, records, help, and non-web map/camera destinations. Multiple flows are incomplete or defective; see the completed I1 evidence report for classifications. This reference does not elevate those flows to a stable contract.
